# Default Documentation Routing

Use this only when the repository does not define a more specific policy.

| Information | Default destination |
| --- | --- |
| Durable project purpose and boundaries | `PROJECT.md` |
| Adopted behavior and constraints | `docs/spec/`, `docs/product/`, `docs/arch/`, or `docs/data/` |
| Adopted operating procedure | `docs/guide/` or `docs/ops/` |
| Candidate, comparison, raw research | `docs/candi-ref/` |
| Implementation work and progress | `docs/imp/imp-*` |
| User decision or confirmation | `docs/imp/user-judge.md` |
| User-owned action | `docs/imp/user-tasks.md` |
| Completed work and verification | `docs/imp/imp-comp.md` |
| Session recovery context | `docs/diary/` or designated handoff file |
| Reusable cross-project knowledge | central vault after its policy gate |

For each event, update only the destinations whose facts changed.
